# Обновление: Фильтры HH.ru для кандидатов из Huntflow

## ✅ Выполненные изменения

### 1. Убрана обязательность ID вакансии на HH.ru
**Изменения:**
- Страница автоматически загружает всех кандидатов из Huntflow для вакансии
- Не требуется указывать ID вакансии на HH.ru для просмотра
- Форма импорта сделана опциональной (collapsible)

### 2. Добавлена форма фильтров HH.ru
**Файл:** `templates/huntflow/hh_responses_list.html`

**Фильтры:**
- Минимальный/максимальный возраст
- Минимальный/максимальный опыт работы (в годах)
- ID городов (через запятую)
- Пол (любой/мужской/женский)

**Применение:**
- Фильтры применяются к кандидатам из Huntflow
- Используются данные из HHResponse (если есть связь) или из самого applicant
- Если данных нет - кандидат проходит фильтр (не отклоняется)

### 3. Обновлена логика фильтрации в View
**Файл:** `apps/huntflow/views.py`

**Изменения:**
- Фильтры применяются ко всем кандидатам из Huntflow
- Используются данные из разных источников:
  - Из HHResponse (если есть связь)
  - Из данных applicant из Huntflow API
- Показывается статус прохождения фильтров для каждого кандидата

### 4. Обновлена статистика
**Новые показатели:**
- Всего кандидатов
- Из HH.ru
- Прошли фильтры HH.ru
- Не прошли фильтры HH.ru

### 5. Обновлена таблица кандидатов
**Добавлено:**
- Индикатор прохождения фильтров HH.ru (зеленый/желтый badge)
- Причины отклонения фильтрами (если не прошел)
- Индикатор источника (HH.ru или другой)

## 🎯 Новый пользовательский сценарий

### Шаг 1: Открытие страницы
1. Пользователь переходит на `/huntflow/accounts/<account_id>/hh/vacancies/<vacancy_id>/responses/`
2. **Автоматически** загружаются все кандидаты из Huntflow для данной вакансии
3. Показывается статистика по кандидатам

### Шаг 2: Применение фильтров HH.ru
1. Пользователь настраивает фильтры:
   - Возраст (мин/макс)
   - Опыт работы (мин/макс в годах)
   - Локация (ID городов через запятую)
   - Пол
2. Нажимает "Применить фильтры"
3. Страница обновляется с примененными фильтрами

### Шаг 3: Просмотр результатов
1. Кандидаты отображаются с индикаторами:
   - ✅ Прошел фильтры HH.ru (зеленый badge)
   - ⚠️ Не прошел фильтры (желтый badge с причинами)
2. Можно фильтровать по статусу фильтрации:
   - Все
   - Прошли фильтры
   - Не прошли фильтры

### Шаг 4: Импорт новых откликов (опционально)
1. Если нужно импортировать новые отклики из HH.ru:
   - Раскрыть форму "Импорт новых откликов из HH.ru"
   - Указать ID вакансии на HH.ru
   - Нажать "Импортировать"
2. Фильтры из основной формы будут применены автоматически

## 📊 Логика фильтрации

### Источники данных для фильтрации

1. **Возраст:**
   - Из `HHResponse.birth_date` (если есть связь)
   - Из `applicant.birthday` (если есть в Huntflow API)
   - Если данных нет - проходит фильтр

2. **Опыт работы:**
   - Из `HHResponse.experience_json` (если есть связь)
   - Если данных нет - проходит фильтр

3. **Локация:**
   - Из `HHResponse.location_id` (если есть связь)
   - Из `applicant.area.id` (если есть в Huntflow API)
   - Если данных нет - проходит фильтр

4. **Пол:**
   - Из `HHResponse.gender` (если есть связь)
   - Из `applicant.gender.id` (если есть в Huntflow API)
   - Если данных нет - проходит фильтр

### Правила фильтрации

- Если фильтр не задан - все кандидаты проходят
- Если данных для проверки нет - кандидат проходит фильтр
- Если данные есть и не соответствуют - кандидат не проходит

## 🔧 Технические детали

### GET параметры для фильтров
- `min_age` - минимальный возраст
- `max_age` - максимальный возраст
- `min_experience_years` - минимальный опыт
- `max_experience_years` - максимальный опыт
- `allowed_locations` - ID городов через запятую
- `allowed_genders` - пол (any/male/female)
- `source` - источник (hh/other)
- `filter_status` - статус фильтрации (passed/rejected)
- `count` - количество на странице
- `page` - номер страницы

### Сохранение фильтров
- Фильтры сохраняются в URL параметрах
- При пагинации фильтры сохраняются
- Можно сбросить фильтры кнопкой "Сбросить фильтры"

## ✨ Преимущества

1. **Автоматическая загрузка** - не нужно указывать ID вакансии на HH.ru
2. **Гибкая фильтрация** - фильтры применяются к существующим кандидатам
3. **Понятная индикация** - видно, кто прошел фильтры, а кто нет
4. **Опциональный импорт** - можно импортировать новые отклики при необходимости
5. **Сохранение фильтров** - фильтры сохраняются в URL
