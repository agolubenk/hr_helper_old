# Исправление: Фильтрация по статусу "отклик"

## ✅ Проблема

Подтягивались все кандидаты, которые уже есть на этапах (статусах), а не только отклики. Нужно было получать только кандидатов со статусом "отклик".

## ✅ Решение

Добавлена фильтрация кандидатов по статусу "отклик" при получении данных из Huntflow API.

### Изменения в `views.py`

1. **Получение статусов вакансии:**
```python
vacancy_statuses = huntflow_service.get_vacancy_statuses(account_id)
```

2. **Поиск статуса "отклик":**
   - Ищем статус с `order=1` (обычно первый статус)
   - Или статус с названием, содержащим: "отклик", "response", "new", "новая"

3. **Фильтрация кандидатов:**
```python
applicants_params = {
    'vacancy': vacancy_id,
    'page': page,
    'count': count
}

# Добавляем фильтр по статусу, если нашли статус отклика
if response_status_id:
    applicants_params['status'] = response_status_id
```

### Логика поиска статуса

1. Проверяем все статусы вакансии
2. Ищем статус с `order=1` или название содержит ключевые слова
3. Если не нашли - берем первый статус из списка
4. Используем найденный статус для фильтрации кандидатов

### Отображение в шаблоне

Добавлена информация о том, какой статус используется для фильтрации:
```html
{% if response_status_name %}
<br><i class="fas fa-filter me-1"></i>Фильтр: статус "{{ response_status_name }}"
{% endif %}
```

## 🔧 Технические детали

### API Endpoint
- `GET /accounts/{account_id}/applicants?vacancy={vacancy_id}&status={status_id}`

### Параметры фильтрации
- `vacancy`: ID вакансии (обязательно)
- `status`: ID статуса "отклик" (добавлено)
- `page`: номер страницы
- `count`: количество на странице (максимум 30)

### Логирование
- Логируется найденный статус отклика
- Логируется использование первого статуса, если не найден специфический

## 📊 Результат

Теперь на странице отображаются только кандидаты со статусом "отклик", а не все кандидаты на всех этапах вакансии.
