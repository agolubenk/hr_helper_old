# Обновление: Автоматическая загрузка откликов из Huntflow

## ✅ Выполненные изменения

### 1. Обновлен View `hh_responses_list`
**Файл:** `apps/huntflow/views.py`

**Изменения:**
- Теперь получает кандидатов напрямую из Huntflow API для данной вакансии
- Использует метод `get_applicants()` с параметром `vacancy`
- Связывает кандидатов с данными из HH.ru (если есть связь в БД)
- Отображает всех кандидатов вакансии, а не только импортированных из HH.ru

**Ключевые особенности:**
- Автоматическая загрузка всех кандидатов при открытии страницы
- Не требуется указывать ID вакансии на HH.ru для просмотра
- Показывает всех кандидатов вакансии с индикацией источника

### 2. Обновлен шаблон `hh_responses_list.html`
**Файл:** `templates/huntflow/hh_responses_list.html`

**Изменения:**
- Обновлена статистика: "Всего кандидатов", "Из HH.ru", "Другие источники"
- Обновлена таблица для отображения кандидатов из Huntflow API
- Добавлены индикаторы источника (HH.ru или другой)
- Обновлены фильтры: фильтр по источнику вместо статуса импорта
- Обновлена пагинация для работы с данными из API

### 3. Обновлена логика сохранения в БД
**Файл:** `apps/huntflow/hh_integration.py`

**Изменения:**
- Добавлен метод `_save_hh_response_to_db()` для сохранения откликов в БД
- При импорте создается связь между кандидатом (applicant_id) и HHResponse
- Сохраняется информация о вакансии на HH.ru (hh_vacancy_id)
- Обновляется существующий отклик, если он уже есть в БД

### 4. Обновлена связь данных
**Файл:** `apps/huntflow/views.py`

**Изменения:**
- При загрузке страницы происходит связывание кандидатов из Huntflow с данными из HH.ru
- Используется поле `applicant_id` в модели `HHResponse` для связи
- Кандидаты обогащаются информацией из HH.ru (ссылка на резюме, ID вакансии на HH.ru)

## 🎯 Новый пользовательский сценарий

### Шаг 1: Открытие страницы
1. Пользователь переходит на `/huntflow/accounts/<account_id>/hh/vacancies/<vacancy_id>/responses/`
2. **Автоматически** загружаются все кандидаты из Huntflow для данной вакансии
3. Не требуется указывать ID вакансии на HH.ru

### Шаг 2: Просмотр кандидатов
1. Пользователь видит всех кандидатов вакансии
2. Кандидаты из HH.ru помечены зеленым индикатором
3. Показана статистика: всего, из HH.ru, другие источники

### Шаг 3: Фильтрация (опционально)
1. Пользователь может отфильтровать по источнику:
   - Все источники
   - Только из HH.ru
   - Только другие источники

### Шаг 4: Импорт новых откликов (опционально)
1. Если нужно импортировать новые отклики из HH.ru:
   - Указывает ID вакансии на HH.ru
   - Настраивает фильтры
   - Запускает импорт
2. После импорта страница автоматически обновляется

## 📊 Структура данных

### Кандидаты из Huntflow API
```python
{
    'applicant': {
        'id': 123,
        'first_name': 'Иван',
        'last_name': 'Иванов',
        'email': 'ivan@example.com',
        'phone': '+79261234567',
        'position': 'Разработчик',
        'status': {...},
        'created': '2024-12-01T10:00:00'
    },
    'hh_response': HHResponse объект или None,
    'is_from_hh': True/False,
    'hh_vacancy_id': '87654321' или None,
    'hh_resume_url': 'https://hh.ru/resume/...' или None
}
```

### Связь данных
- `HHResponse.applicant_id` → связь с кандидатом в Huntflow
- При импорте создается/обновляется запись в `HHResponse`
- При просмотре происходит поиск связи по `applicant_id`

## 🔧 Технические детали

### API Endpoints
- **Получение кандидатов:** `GET /accounts/<account_id>/applicants?vacancy=<vacancy_id>`
- **Импорт откликов:** `POST /huntflow/accounts/<account_id>/hh/vacancies/<vacancy_id>/import/`

### Пагинация
- Используется пагинация Huntflow API (0-based)
- Параметры: `page` (начиная с 1), `count` (20/50/100)

### Фильтрация
- По источнику: `?source=hh` или `?source=other`
- По количеству на странице: `?count=50`

## ✨ Преимущества

1. **Автоматическая загрузка** - не нужно указывать ID вакансии на HH.ru
2. **Все кандидаты** - видно всех кандидатов вакансии, не только из HH.ru
3. **Индикация источника** - понятно, откуда пришел кандидат
4. **Гибкая фильтрация** - можно фильтровать по источнику
5. **Сохранение связи** - при импорте сохраняется связь с HH.ru данными
