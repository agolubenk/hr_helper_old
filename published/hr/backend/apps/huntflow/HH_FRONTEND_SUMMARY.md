## 🎯 Пользовательский сценарий

### Шаг 1: Переход с Dashboard
1. Пользователь заходит на `/huntflow/`
2. Видит кнопку "Работа с откликами из HH.ru"
3. Нажимает на кнопку

### Шаг 2: Выбор вакансии
1. Открывается страница `/huntflow/accounts/<account_id>/hh/vacancies/`
2. Пользователь видит список всех вакансий организации
3. Вакансии с настроенной синхронизацией помечены зеленым индикатором
4. Пользователь выбирает вакансию и нажимает "Просмотр откликов"

### Шаг 3: Просмотр и импорт откликов
1. Открывается страница `/huntflow/accounts/<account_id>/hh/vacancies/<vacancy_id>/responses/`
2. Пользователь видит:
   - Информацию о вакансии
   - Статистику по откликам (всего, импортировано, отфильтровано, ожидает)
   - Форму импорта откликов с фильтрами
   - Список уже импортированных откликов (если есть)
3. Для импорта новых откликов:
   - Указывает ID вакансии на HH.ru
   - Настраивает фильтры (возраст, опыт, локация, пол)
   - Нажимает "Импортировать отклики"
4. Система:
   - Получает отклики из HH.ru API
   - Применяет фильтры
   - Импортирует прошедших фильтр кандидатов в Huntflow
   - Сохраняет все отклики в БД
   - Обновляет страницу с результатами

### Шаг 4: Фильтрация откликов
1. Пользователь может фильтровать отображение откликов по статусу:
   - Все статусы
   - Импортировано
   - Отфильтровано
   - Ожидает обработки
   - Ошибка

## 📋 Функциональность

### Форма импорта откликов
- **Поля:**
  - ID вакансии на HH.ru (обязательное)
  - Минимальный возраст (по умолчанию: 18)
  - Максимальный возраст (по умолчанию: 65)
  - Минимальный опыт в годах (по умолчанию: 1)
  - Максимальный опыт в годах (по умолчанию: 50)
  - ID городов (через запятую, например: 1,2,3)
  - Пол (любой/мужской/женский)
  - Проверка дубликатов (включена по умолчанию)

### Таблица откликов
- **Колонки:**
  - Кандидат (ФИО)
  - Контакты (email, телефон)
  - Локация
  - Возраст
  - Опыт работы
  - Статус импорта
  - Дата отклика
  - Действия (ссылка на резюме на HH.ru)

### Статистика
- Всего откликов
- Импортировано
- Отфильтровано
- Ожидает обработки

## 🔧 Технические детали

### API Endpoints
- **Импорт откликов:** `POST /huntflow/accounts/<account_id>/hh/vacancies/<vacancy_id>/import/`
  - Параметры: `hh_vacancy_id`, `filters` (JSON)
  - Возвращает: результаты импорта (количество импортированных, отфильтрованных, ошибок)

### Модели данных
- `HHResponse` - хранит отклики из HH.ru
- `HHSyncConfiguration` - конфигурации синхронизации
- `HHSyncLog` - логи синхронизаций
- `HHFilterStatistics` - статистика фильтрации

### Обработка ошибок
- Валидация обязательных полей
- Обработка ошибок API
- Отображение понятных сообщений пользователю
- Логирование всех операций
