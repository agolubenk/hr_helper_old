# Приложение отчетности

Приложение для генерации отчетов по скринингам и интервью на основе данных календаря Google Calendar.

## Возможности

- **Отчеты по компании** - общая статистика по всем вакансиям и рекрутерам
- **Отчеты по рекрутеру** - статистика по каждому рекрутеру
- **Отчеты по вакансии** - статистика по конкретной вакансии
- **Отчеты по интервьюеру** - статистика по каждому интервьюеру

## Периоды отчетности

- Ежедневная
- Понедельная
- Помесячная
- Поквартальная
- Годовая

## Как это работает

### Определение типа интервью

Система определяет тип интервью (скрининг или интервью) на основе заголовков событий календаря:

1. **Скрининги** - определяются по полю `invite_title` из модели `Vacancy`
2. **Интервью** - определяются по полю `tech_invite_title` из модели `Vacancy`

### Определение вакансии

Вакансия определяется путем сопоставления заголовка события календаря с заголовками инвайтов в вакансиях. Система использует нормализацию заголовков (приведение к нижнему регистру, удаление лишних пробелов) для более точного сопоставления.

### Источник данных

Все данные берутся из Google Calendar через Google Calendar API. Система анализирует события календаря пользователя и определяет:
- Тип события (скрининг/интервью)
- Вакансию
- Рекрутера
- Интервьюера (по участникам события)

## Структура

### Модели

- `ReportCache` - кэш для хранения предвычисленных отчетов (опционально)

### Сервисы

- `CalendarEventAnalyzer` - анализатор календарных событий
- `ReportGenerator` - генератор отчетов

### Views

- `report_dashboard` - главная страница отчетности
- `company_report` - отчет по компании
- `recruiter_report` - отчет по рекрутеру
- `vacancy_report` - отчет по вакансии
- `interviewer_report` - отчет по интервьюеру
- `api_report_data` - API endpoint для получения данных отчета в JSON

## URL маршруты

- `/reporting/` - главная страница
- `/reporting/company/` - отчет по компании
- `/reporting/recruiter/` - список рекрутеров
- `/reporting/recruiter/<id>/` - отчет по рекрутеру
- `/reporting/vacancy/` - список вакансий
- `/reporting/vacancy/<id>/` - отчет по вакансии
- `/reporting/interviewer/` - список интервьюеров
- `/reporting/interviewer/<id>/` - отчет по интервьюеру
- `/reporting/api/data/` - API endpoint

## Использование

### Параметры фильтров

Все отчеты поддерживают следующие параметры GET-запроса:

- `period` - период группировки (`daily`, `weekly`, `monthly`, `quarterly`, `yearly`)
- `start_date` - дата начала периода (формат: `YYYY-MM-DD`)
- `end_date` - дата окончания периода (формат: `YYYY-MM-DD`)

### Примеры

```
/reporting/company/?period=monthly&start_date=2024-01-01&end_date=2024-12-31
/reporting/recruiter/1/?period=weekly&start_date=2024-01-01&end_date=2024-12-31
/reporting/vacancy/5/?period=daily&start_date=2024-01-01&end_date=2024-12-31
```

## Требования

- Django 4.2+
- Google OAuth аккаунт пользователя должен быть подключен
- Доступ к Google Calendar API
- Настроенные вакансии с заполненными полями `invite_title` и `tech_invite_title`

## Зависимости

- `apps.google_oauth` - для доступа к Google Calendar API
- `apps.vacancies` - для получения информации о вакансиях
- `apps.interviewers` - для получения информации об интервьюерах
- `python-dateutil` - для работы с периодами

## Ограничения

1. Система определяет вакансию по точному совпадению заголовков. Если заголовок события не совпадает с заголовками инвайтов в вакансиях, событие будет помечено как `unknown`.

2. Интервьюер определяется по email участников события. Если интервьюер не найден в системе или его email не совпадает, поле `interviewer` будет `None`.

3. Данные берутся только из календаря пользователя (`primary`). Для использования календаря компании необходимо настроить соответствующий календарь в настройках.
