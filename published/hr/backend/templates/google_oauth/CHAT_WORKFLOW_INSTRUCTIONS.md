# Инструкция по использованию чата HR-помощника (chat_workflow)

Данная инструкция описывает работу чата на странице `/google-oauth/chat/` — единого экрана рекрутера для HR-скрининга, планирования Tech Screening и Final Interview, а также добавления кандидатов на вакансию.

## 1. Загрузка файла с резюме для добавления кандидата на вакансию

### Как это работает

1. **Выберите вакансию** в селекторе в правой части экрана (обязательно — без выбранной вакансии добавление кандидата не работает).
2. **Прикрепите файл** — нажмите кнопку со значком скрепки (📎) слева от поля ввода сообщения.
3. **Поддерживаемые форматы**: PDF, DOC, DOCX, TXT.
4. После выбора файл автоматически отправляется на парсинг в Huntflow.
5. При успешном парсинге кандидат автоматически создаётся и добавляется **на текущую выбранную вакансию**.

### Важно

- Кандидат всегда добавляется на **вакансию, выбранную в селекторе** на момент загрузки файла.
- Перед загрузкой обязательно убедитесь, что в селекторе выбрана нужная вакансия.
- Файл отправляется на `/huntflow/parse-resume-file/`; при ошибке парсинга появится уведомление.
- В чате отображается статус: «Создание кандидата…», затем «Данные сохранены и переданы в Huntflow».

### Пример

1. Выберите вакансию «Backend Python Developer».
2. Нажмите кнопку прикрепления файла и выберите `резюме_иванов.pdf`.
3. Дождитесь обработки — кандидат будет создан и привязан к выбранной вакансии.

## 2. Команды /s, /t и /in

В чате **всегда нужно указывать явную команду**. Без команды система выведет подсказку:

> Укажи команду: /s для HR-скрининга, /t для Tech Screening или /in для Final Interview

### Команда /s — HR-скрининг

**Назначение:** запуск HR-скрининга (оценка кандидата).

**Синтаксис:**
```
/s <текст>
```

**Примеры:**
- `/s https://huntflow.ru/my/org#/vacancy/123/filter/workon/id/789`
- `/s Кандидат Иванов, backend Python. Посмотри опыт с Django, Celery, PostgreSQL.`

**Синоним:** `/hr` (работает так же).

---

### Команда /t — Tech Screening

**Назначение:** создание инвайта на техническое интервью.

**Синтаксис:**
```
/t <текст>
```

**Примеры:**
- `/t https://huntflow.ru/.../id/789 завтра 14:00`
- `/t Назначь техскрин кандидату Иванов на следующей неделе во второй половине дня`

---

### Команда /in — Final Interview

**Назначение:** создание инвайта на финальное интервью.

**Синтаксис:**
```
/in <текст>
```

**Примеры:**
- `/in https://huntflow.ru/.../id/789 пятница 16:00`
- `/in Финал для Иванова: согласуй время на следующей неделе`

**Синонимы:** `/invite`, `/inv` (работают так же).

---

### Особенности ввода команд

| Аспект | Описание |
|--------|----------|
| Пробел после команды | Команда распознаётся и с пробелом (`/s `), и без пробела (`/s` в конце строки). Регулярное выражение: `^/s(\s|$)` |
| Регистр | Регистр не важен — `/S`, `/s`, `/IN` и т.д. работают одинаково |
| Русская раскладка | Команды, набранные в русской раскладке, распознаются так же: `/ы` = `/s`, `/е` = `/t`, `/шт` = `/in`, `/рк` = `/hr`, `/вуд` = `/del`, `/фвв` = `/add`, `/шм` = `/inv` |
| Позиция | Команда должна находиться **в начале сообщения** |
| Удаление команды | Текст команды не показывается в bubble сообщения; отображается только содержимое после команды |

## 3. Специфика ссылок и переключение между вакансиями

### Форматы ссылок Huntflow

1. **С вакансией в URL:**
   ```
   https://huntflow.ru/my/org#/vacancy/123/filter/workon/id/789
   ```
   - `123` — ID вакансии
   - `789` — ID кандидата

2. **Без вакансии (только кандидат):**
   ```
   https://huntflow.ru/my/org#/applicants/filter/all/77231621
   ```
   - `77231621` — ID кандидата
   - Вакансия определяется через Huntflow API по данным кандидата

### Переключение между вакансиями

Если вы находитесь в чате по вакансии A и отправляете сообщение со ссылкой на кандидата по **другой** вакансии B:

1. Система определяет вакансию из ссылки (или по данным кандидата в Huntflow).
2. Если вакансия B отличается от текущей, происходит **автоматическое переключение** на чат вакансии B.
3. Ваше сообщение сохраняется в сессию чата, соответствующую вакансии B.
4. Сообщение отображается в контексте правильной вакансии.

Таким образом, можно отправлять ссылки на кандидатов с других вакансий — система сама переключит контекст.

## 4. Выбор интервьюеров и формата для финального интервью

### Выбор интервьюеров

1. **Через упоминание `@`**  
   Введите `@` в поле сообщения при команде `/in` — появится выпадающий список интервьюеров.

2. **Навигация:**
   - `↑` / `↓` — переключение по списку
   - `Enter` — выбор выделенного интервьюера
   - `Escape` — закрыть список

3. **Результат:** в текст подставляется `@username` интервьюера, его данные отправляются на сервер в скрытом поле `selected_interviewer`.

### Выбор формата (онлайн / офис)

Для **Final Interview** (`/in`) доступны два формата:

1. **Скрининг** (кнопка по умолчанию) — без выбора интервьюеров и формата.
2. **Интервью** — при нажатии на кнопку «Интервью» появляются:
   - **Онлайн** (по умолчанию) — встреча в Google Meet
   - **Офис** — встреча в офисе (адрес берётся из настроек компании)

Выбор сохраняется в скрытом поле `interview_format` (значения: `online` или `office`).

### Важно

- Если интервьюеры **не выбраны** (ни через `@`, ни через UI), в календарное событие **не добавляется никто** из списка интервьюеров вакансии.
- Выбранные интервьюеры подставляются в событие календаря как участники (attendees).

## 5. Подтверждение отказа по офису и финансам

После HR-скрининга (`/s`) система может показать форму отказа, если:

1. **Отказ по офису** — кандидат не подходит под офисный формат (настройки вакансии/скрининга).
2. **Отказ по финансам** — зарплатные ожидания кандидата выше вилки по вакансии.

### Как выглядит форма

Под сообщением HR-скрининга появляется блок:
- «Отказать кандидату?»
- Кнопка **«Да»** — выполнить отказ (статус в Huntflow меняется на отказ).
- Кнопка **«Нет»** — не отказывать.

### Дополнительные действия

- При отказе по офису доступна кнопка копирования шаблона отказа (иконка ✕).
- При превышении зарплаты — кнопка копирования шаблона «Финансы — больше».

Ответ сохраняется в метаданные сообщения (`rejection_form_answered`, `rejection_form_answer`).

## 6. Изменение времени проведения встречи (в скобках)

Для инвайтов (`/t`, `/in`) можно указать **кастомную длительность** встречи **в скобках после даты и времени**.

### Поддерживаемые форматы в скобках

| Формат | Пример |
|--------|--------|
| Часы (полностью) | `(1 час)`, `(2 часа)` |
| Минуты (полностью) | `(30 минут)`, `(45 минут)` |
| Сокращения | `(1 ч)`, `(2 ч)`, `(30 мин)`, `(45 м)` |
| Полчаса | `(полчаса)` |

### Примеры

```
/in https://huntflow.ru/.../id/789 2025-09-15 14:00 (1 час)
/t https://huntflow.ru/.../id/789 пятница 16:00 (30 минут)
/in https://huntflow.ru/.../id/789 завтра 15:00 (полчаса)
```

### Обработка опечаток

Система пытается исправить типичные опечатки раскладки:
- `xfc` → `час`
- `vfc` → `час`
- `vf` → `час`

## 7. Поддерживаемые ошибочные вводы и форматы данных

### Дата и время

#### Даты

| Тип | Примеры |
|-----|---------|
| Числовые | `10.10.2024`, `10/10/2024`, `10-10-2024`, `2024-10-10`, `10.10` |
| Текстовые месяцы | `10 октября`, `10 oct`, `15 октября 2024` |
| Дни недели | `понедельник`, `пн`, `monday`, `mon` |
| Относительные | `сегодня`, `завтра`, `послезавтра`, `через неделю`, `next week` |

#### Время

| Тип | Примеры |
|-----|---------|
| Полный формат | `15:00`, `15.30`, `15-45` |
| Только час | `15`, `15 ч`, `15 час` |
| С предлогами | `в 16:30`, `к 14:00`, `после 17` |
| После дня недели | `пн 15`, `вторник в 14` |

#### Неправильная раскладка клавиатуры

Автоматически исправляются варианты вроде:
- `GY` → `ПН` (понедельник)
- `ds` → `вт` (вторник)
- `pfdnhf` → `завтра`

### Валидация и коррекция

- **Бизнес-часы:** 11:00–17:45 (из настроек пользователя или по умолчанию).
- **Временные слоты:** минуты округляются до 00, 15, 30, 45.
- **Прошедшие даты** отклоняются.
- **Время вне рабочих часов** переносится в допустимый диапазон.
- **Диапазон дат:** не более 90 дней вперёд.

### Ссылки

Поддерживаются форматы:
- С вакансией: `/vacancy/123/filter/456/id/789`
- Без вакансии: `/applicants/filter/all/77231621`

### Команды — альтернативные варианты

| Основная | Альтернативы |
|----------|--------------|
| `/s` | `/hr`, `/screen` |
| `/in` | `/invite`, `/inv` |

Команды учитываются с пробелом и без (`/s `, `/s\n` и т.п.).

## Быстрый справочник

| Действие | Команда / способ |
|----------|------------------|
| HR-скрининг | `/s` или `/hr` + текст/ссылка |
| Tech Screening | `/t` + текст/ссылка + дата/время |
| Final Interview | `/in` + текст/ссылка + дата/время |
| Добавить кандидата по файлу | Выбрать вакансию → прикрепить PDF/DOC/DOCX/TXT |
| Указать интервьюера | `@` + имя в команде `/in` |
| Кастомная длительность | Дата и время + `(1 час)` или `(30 минут)` |
| Подтвердить отказ | Кнопки «Да» / «Нет» в форме под HR-скринингом |
