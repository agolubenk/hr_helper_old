## 📖 Обзор

Chrome расширение **HRHelper LinkedIn → Huntflow** позволяет:
- ✅ Добавлять кнопку "Huntflow" на профилях LinkedIn
- ✅ Сохранять связи между LinkedIn профилями и кандидатами в Huntflow
- ✅ Быстро открывать кандидатов в Huntflow прямо из LinkedIn
- ✅ Отслеживать статус кандидата (сохранен ли в Huntflow)

---

## 🔧 Подготовка к установке

### Требования

1. **Chrome браузер** (версия 88+)
2. **Доступ к HRHelper** (должны быть залогинены)
3. **API токен** из HRHelper

### Необходимая информация

Перед настройкой вам понадобится:
- 🌐 **URL продакшен сервера** (например: `https://hrhelper.example.com`)
- 🔑 **API токен** (получается в HRHelper)

## ⚙️ Настройка для продакшена

### Шаг 1: Обновление manifest.json

Перед установкой расширения нужно добавить разрешения для вашего продакшен хоста.

Откройте файл `manifest.json` и обновите секцию `host_permissions`:

```json
{
  "manifest_version": 3,
  "name": "HRHelper LinkedIn → Huntflow",
  "version": "0.1.0",
  "description": "Кнопка Huntflow на профилях LinkedIn: открыть кандидата или сохранить в Huntflow через HRHelper.",
  "permissions": ["storage"],
  "host_permissions": [
    "https://www.linkedin.com/*",
    "https://calendar.google.com/*",
    "https://meet.google.com/*",
    "http://localhost:8000/*",
    "http://127.0.0.1:8000/*",
    "https://your-production-domain.com/*"  // ← ДОБАВЬТЕ ВАШ ДОМЕН
  ],
  // ... остальная конфигурация
}
```

**Важно:** Замените `your-production-domain.com` на реальный домен вашего продакшен сервера.

**Примеры:**
```json
"https://hrhelper.example.com/*"
"https://app.hrhelper.io/*"
"https://hrhelper.company.com/*"
```

### Шаг 2: Проверка CORS настроек

Убедитесь, что на вашем продакшен сервере настроен CORS для работы с расширением.

В файле `config/settings.py` или `config/settings_production.py` должны быть настройки:

```python
CORS_ALLOWED_ORIGINS = [
    "chrome-extension://your-extension-id",  # Будет известен после установки
    # Или используйте:
    # "chrome-extension://*",  # Разрешить все расширения (менее безопасно)
]

# Для разработки можно использовать:
CORS_ALLOW_ALL_ORIGINS = False  # В продакшене должно быть False
```

**Примечание:** Extension ID будет известен после установки расширения в Chrome.

## 📦 Установка расширения

### Вариант 1: Установка из исходников (для разработки)

1. **Откройте Chrome** и перейдите в `chrome://extensions/`

2. **Включите режим разработчика** (Developer mode) - переключатель в правом верхнем углу

3. **Нажмите "Load unpacked"** (Загрузить распакованное)

4. **Выберите папку** с расширением:
   ```
   HR_Helper/chrome-extension/hrhelper-linkedin-huntflow/
   ```

5. **Расширение установлено!** Вы увидите его в списке расширений

6. **Скопируйте Extension ID** - он понадобится для настройки CORS
   - ID будет в формате: `abcdefghijklmnopqrstuvwxyz123456`

### Вариант 2: Упаковка для распространения (для продакшена)

1. **Откройте Chrome** и перейдите в `chrome://extensions/`

2. **Включите режим разработчика**

3. **Нажмите "Pack extension"** (Упаковать расширение)

4. **Выберите папку** с расширением

5. **Создайте приватный ключ** (или используйте существующий)

6. **Получите `.crx` файл** - его можно распространять

## ⚙️ Настройка расширения

### Шаг 1: Открытие настроек

1. **Кликните на иконку расширения** в панели инструментов Chrome
2. Или **правый клик на иконку** → **"Options"** (Настройки)
3. Или перейдите в `chrome://extensions/` → найдите расширение → **"Options"**

### Шаг 2: Заполнение настроек

В форме настроек заполните:

#### Base URL HRHelper
```
https://your-production-domain.com
```

**Примеры:**
- `https://hrhelper.example.com`
- `https://app.hrhelper.io`
- `https://hrhelper.company.com`

**Важно:**
- ✅ Используйте `https://` для продакшена
- ✅ Не добавляйте `/` в конце URL
- ✅ Убедитесь, что URL доступен из браузера

#### API Token (DRF)
```
your-api-token-here
```

**Как получить токен** - см. раздел [Получение API токена](#получение-api-токена)

### Шаг 3: Сохранение настроек

1. **Нажмите кнопку "Сохранить"**
2. **Проверьте сообщение** - должно появиться "Сохранено." зеленым цветом
3. **Если ошибка** - проверьте правильность URL и токена

## 🔑 Получение API токена

### Способ 1: Через веб-интерфейс (рекомендуется)

1. **Откройте HRHelper** в браузере и **войдите в систему**

2. **Перейдите по адресу:**
   ```
   https://your-production-domain.com/api/v1/accounts/users/token/
   ```

3. **Если вы не авторизованы**, вас перенаправит на страницу входа

4. **После входа** вы увидите JSON ответ:
   ```json
   {
     "token": "your-api-token-here-very-long-string"
   }
   ```

5. **Скопируйте значение `token`** и вставьте в настройки расширения

### Способ 2: Через API запрос (для разработчиков)

Используйте curl или Postman:

```bash
curl -X POST https://your-production-domain.com/api/v1/accounts/users/token/ \
  -H "Content-Type: application/json" \
  -d '{"username": "your-username", "password": "your-password"}'
```

**Ответ:**
```json
{
  "token": "your-api-token-here"
}
```

### Способ 3: Через Django admin (для администраторов)

1. **Откройте Django Admin:** `https://your-production-domain.com/admin/`

2. **Перейдите в:** `Authtoken` → `Tokens`

3. **Создайте новый токен** для нужного пользователя

4. **Скопируйте токен** и используйте в расширении

## 🐛 Устранение проблем

### Проблема 1: Кнопка не появляется на LinkedIn

**Возможные причины:**

1. **Неправильный API Token**
   - ✅ Проверьте токен в настройках
   - ✅ Получите новый токен через `/api/v1/accounts/users/token/`

2. **Неправильный Base URL**
   - ✅ Проверьте, что URL доступен из браузера
   - ✅ Убедитесь, что используется `https://` для продакшена
   - ✅ Проверьте, что нет `/` в конце URL

3. **CORS ошибки**
   - ✅ Откройте консоль браузера (F12)
   - ✅ Проверьте ошибки CORS в Console или Network tab
   - ✅ Убедитесь, что Extension ID добавлен в `CORS_ALLOWED_ORIGINS`

4. **Расширение не загружено**
   - ✅ Проверьте `chrome://extensions/` - расширение должно быть включено
   - ✅ Перезагрузите страницу LinkedIn

**Решение:**
```javascript
// Включите DEBUG режим в content.js (временно)
const DEBUG = true;

// Перезагрузите расширение и проверьте консоль
```

### Проблема 2: Ошибка "Нет API токена"

**Причина:** Токен не указан или не сохранен

**Решение:**
1. Откройте настройки расширения
2. Убедитесь, что токен вставлен
3. Нажмите "Сохранить"
4. Проверьте сообщение "Сохранено."

### Проблема 3: Ошибка CORS

**Ошибка в консоли:**
```
Access to fetch at 'https://...' from origin 'chrome-extension://...' has been blocked by CORS policy
```

**Решение:**

1. **Получите Extension ID:**
   - Откройте `chrome://extensions/`
   - Найдите расширение
   - Скопируйте ID (например: `abcdefghijklmnopqrstuvwxyz123456`)

2. **Обновите настройки CORS в Django:**
   ```python
   # config/settings_production.py
   CORS_ALLOWED_ORIGINS = [
       f"chrome-extension://{EXTENSION_ID}",
       # Или для всех расширений (менее безопасно):
       # "chrome-extension://*",
   ]
   ```

3. **Перезапустите Django сервер**

### Проблема 4: Кнопка появляется, но не работает

**Проверьте:**

1. **Откройте консоль браузера (F12)**
2. **Проверьте ошибки в Console tab**
3. **Проверьте Network tab** - должны быть запросы к API

**Возможные ошибки:**
- `401 Unauthorized` → Неправильный API токен
- `404 Not Found` → Неправильный Base URL
- `403 Forbidden` → Проблемы с CORS или правами доступа

### Проблема 5: Расширение не сохраняет настройки

**Решение:**

1. **Проверьте права расширения:**
   - Откройте `chrome://extensions/`
   - Убедитесь, что расширение имеет права "storage"

2. **Очистите и переустановите:**
   - Удалите расширение
   - Перезагрузите Chrome
   - Установите заново

## 📝 Дополнительная информация

### API Endpoints, используемые расширением

1. **GET `/api/v1/huntflow/linkedin-applicants/status/`**
   - Проверка статуса кандидата
   - Параметры: `linkedin_url` (query parameter)
   - Ответ: `{ "exists": true/false, "huntflow_url": "..." }`

2. **POST `/api/v1/huntflow/linkedin-applicants/set-link/`**
   - Сохранение связи LinkedIn → Huntflow
   - Тело запроса: `{ "linkedin_url": "...", "huntflow_url": "..." }`
   - Ответ: `{ "success": true }`

### Безопасность

- ✅ API токены хранятся в `chrome.storage.sync` (синхронизируются между устройствами)
- ✅ Все запросы используют HTTPS в продакшене
- ✅ Токены передаются через заголовок `Authorization: Token ...`

### Обновление расширения

При обновлении расширения:
1. Загрузите новую версию
2. Перезагрузите расширение в `chrome://extensions/`
3. Проверьте настройки (они должны сохраниться)
