Скопируйте каждый блок в соответствующее поле формы. Лимит 1000 символов на поле.

## Обоснование (storage)*

Расширение сохраняет в chrome.storage настройки пользователя: URL сервера HR Helper (baseUrl) и API-токен для авторизации запросов. Эти данные нужны для работы интеграции с системой HR Helper (поиск кандидатов, сохранение связей LinkedIn/Huntflow, смена статусов). Также в storage сохраняются связи «страница резюме (hh.ru, rabota.by) → Huntflow» для отображения данных кандидата на странице резюме. Данные не передаются третьим лицам и используются только для работы расширения.

---

## Обоснование (tabs)*

Разрешение tabs используется для определения активной вкладки при открытии попапа расширения: расширение должно знать, на какой странице находится пользователь (LinkedIn, Google Meet, hh.ru, rabota.by), чтобы показать соответствующий интерфейс (кнопка Huntflow, форма связи резюме–Huntflow, копирование текста по грейду в Meet). Без доступа к вкладкам невозможно корректно предложить действия в контексте текущей страницы.

---

## Обоснование (activeTab)*

activeTab даёт разовый доступ к текущей вкладке только после действия пользователя (клик по иконке расширения). Используется для чтения URL страницы и, при необходимости, контекста (например, ссылка на профиль LinkedIn или страницу резюме) с целью отправить запрос к серверу HR Helper и отобразить в попапе данные кандидата, статус и кнопки действий. Контент страницы не сохраняется и не передаётся никому, кроме указанного пользователем сервера HR Helper.

---

## Обоснование (Разрешение на доступ к хостам)*

Доступ к хостам нужен для единственной цели: интеграция с сервисами, на которых расширение добавляет функциональность, и с сервером HR Helper. LinkedIn (linkedin.com) — отображение кнопки Huntflow и сохранение связи профиль–кандидат. Google Calendar и Google Meet (calendar.google.com, meet.google.com) — копирование текста по грейду и ссылок во время созвонов. hh.ru и rabota.by — привязка страницы резюме к кандидату в Huntflow и отображение данных из Huntflow на странице. huntflow.ru / hr.sftntx.com — API-запросы к серверу HR Helper (поиск кандидата, сохранение связей, смена статусов). localhost — только для разработки. Данные с этих сайтов не передаются рекламным сетям и используются только для заявленных функций расширения.

## Удалённый код

**Рекомендация:** выберите вариант **«Нет, я не использую разрешение "удаленный код"»**.

В пакете расширения нет загрузки внешнего JavaScript (нет тегов script с внешними URL), нет вызовов eval() и нет загрузки модулей с внешних источников. Весь код входит в пакет расширения. Если в форме по какой-то причине отмечено «Да», переключите на «Нет» и не заполняйте поле обоснования удалённого кода.

## Какие данные собираются (форма «Какие данные вы планируете собирать»)

Рекомендуется отметить только то, что реально используется:

- **Данные для аутентификации** — расширение хранит в браузере введённый пользователем API-токен (в chrome.storage.sync) для доступа к серверу HR Helper. Токен не передаётся третьим лицам.
- **Содержимое сайтов** — на страницах LinkedIn, hh.ru и rabota.by расширение считывает URL страницы и публичные элементы (ссылки на профиль, данные резюме) только для формирования запросов к серверу HR Helper по инициативе пользователя. Данные не продаются и не используются для рекламы.

Остальные категории (медицинская информация, финансовые данные, личная переписка, местоположение и т.д.) расширение не собирает — соответствующие чекбоксы не отмечайте.
