# Руководство пользователя HR Helper

## Установка

1. Скачайте или склонируйте папку расширения.
2. Откройте в Chrome страницу `chrome://extensions/`.
3. Включите **«Режим разработчика»** (переключатель в правом верхнем углу).
4. Нажмите **«Загрузить распакованное расширение»** и выберите папку с расширением.
5. Иконка HR Helper появится на панели расширений (при необходимости закрепите её).

## Настройка

### Открытие настроек

- Правый клик по иконке расширения → **«Параметры»** (или **«Options»**),  
  или откройте попап и нажмите на иконку шестерёнки в заголовке.

### Параметры

| Параметр | Описание |
|----------|----------|
| **Base URL HRHelper** | Адрес сервера HR Helper. По умолчанию: `https://hr.sftntx.com`. Меняйте только если используете свой сервер. |
| **API Token (DRF)** | Токен для доступа к API. Получить можно по ссылке из подсказки под полем (страница выдачи токена на сервере HR Helper). |
| **Тема оформления** | **Светлая** — светлый интерфейс расширения. **Тёмная** — тёмный. **Автоматически** — по теме сайта (LinkedIn, календарь и т.д.) или по настройке темы браузера/системы. |

После ввода данных нажмите **«Сохранить»**. На странице настроек отображается **статус подключения**: успешное подключение к API или сообщение об ошибке (например неверный токен или недоступный сервер).

### Ссылка на интеграции

На странице настроек и в попапе есть ссылка на раздел **«Интеграции»** (или аналог) на сервере HR Helper. Адрес формируется автоматически из Base URL (например `https://hr.sftntx.com/` + путь к интеграциям). Там можно управлять связями и настройками интеграции с Huntflow, LinkedIn и др.

## Поддерживаемые страницы

### LinkedIn

- **Профиль кандидата** (`linkedin.com/in/...`): рядом с кнопками страницы появляется кнопка **«Huntflow»**. По клику открывается попап расширения с данными кандидата из Huntflow: вакансия, статус, кнопки смены статуса, добавления на другую вакансию и т.д.
- **Переписка** (`linkedin.com/messaging/...`): панель с действиями по текущему диалогу и привязка к профилю кандидата в Huntflow.

Оформление виджетов и панели на LinkedIn подстраивается под выбранную тему (светлая/тёмная/авто). При **«Автоматически»** используется тема самой страницы LinkedIn; если её не удаётся определить — тема браузера/системы.

### Huntflow

- **Карточка кандидата** (`huntflow.ru/my/...` или `huntflow.dev/my/...`): плавающее окно с контактами (Telegram, LinkedIn, WhatsApp, Viber и др.) и кнопками копирования: ФИО, телефон, email, ссылка на вакансию. Оформление окна и кнопок зависит от выбранной темы (светлая/тёмная/авто).

### Резюме (hh.ru, rabota.by)

- **Страница резюме** (`hh.ru/resume/...`, `rabota.by/resume/...`): можно привязать ссылку на кандидата в Huntflow и просматривать данные кандидата из Huntflow под заголовком резюме. Плавающее окно с вакансиями и формой сохранения также подстраивается под тему из настроек.

### Google Calendar

- В **событии календаря** (при наличии блока уведомления гостей) отображаются кнопки быстрого контакта (мессенджеры, LinkedIn) и копирование ссылок. Тема кнопок: при **«Автоматически»** — по теме интерфейса календаря (светлая/тёмная), при неопределённости — по теме браузера; при выборе «Светлая» или «Тёмная» используется выбранная тема.

### Google Meet

- Во время **видеозвонка** доступна панель расширения с кнопками: уровень (грейд), Scorecard, контакт в Huntflow и т.д., а также копирование текста и ссылок для удобства работы с кандидатом.

## Темы оформления

- **Светлая** — попап, плавающие окна и кнопки расширения отображаются в светлой теме.
- **Тёмная** — в тёмной теме.
- **Автоматически**:
  - **LinkedIn:** сначала определяется тема страницы LinkedIn (атрибуты, классы, яркость фона); если не удаётся — используется тема браузера (`prefers-color-scheme`).
  - **Календарь:** сначала определяется тема страницы календаря (общий фон страницы, затем область кнопок); при неопределённости — тема браузера.
  - **Попап и остальные страницы:** при неопределённости используется тема браузера/системы.

Настройка темы хранится в расширении и применяется при открытии попапа и при появлении виджетов на страницах.

## Устранение неполадок

| Проблема | Что проверить |
|----------|----------------|
| Нет кнопки «Huntflow» на LinkedIn | Убедитесь, что открыт именно профиль (`/in/...`). Проверьте настройки: указан Base URL и API Token, статус подключения на странице настроек успешен. Перезагрузите страницу. |
| Ошибка при сохранении / «неверный токен» | Проверьте API Token на странице настроек. Получите новый токен по ссылке с страницы HR Helper и вставьте его в настройки. Убедитесь, что Base URL доступен из браузера. |
| Кнопки или попап в «не той» теме | В настройках выберите явно «Светлая» или «Тёмная». При «Автоматически» тема берётся с сайта или из системы — проверьте тему самого сайта (LinkedIn, календарь). |
| Расширение не реагирует после обновления | Обновите страницу (F5 или перезагрузка вкладки). При необходимости переустановите расширение из папки. |
| Нет данных кандидата на резюме / в Huntflow | Убедитесь, что связь с Huntflow настроена (привязка ссылки на резюме, корректный API и токен). Проверьте консоль браузера (F12) на ошибки. |

При проблемах с доступом к API (CORS, сеть) убедитесь, что сервер HR Helper разрешает запросы с вашего домена расширения; при самостоятельном развёртывании сервера см. документацию по настройке CORS и допустимым Extension ID.
