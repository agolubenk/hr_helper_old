# HR Helper — История версий

Ниже — сводка функциональности и изменений по эпохам версий. Актуальная история в интерфейсе настроек расширения (options.html).

---

## Версия 1.2.x (ранее)

Описание функциональности на период 1.2.x.

### Общие возможности

- **Интеграция с HR Helper** — расширение работает через сервер HR Helper (Base URL + API Token). Все данные хранятся на сервере, расширение выступает клиентом.
- **Настройка в попапе** — при отсутствии токена в попапе отображается форма ввода Base URL и API Token с кнопкой «Сохранить». Не нужно переходить на отдельную страницу настроек для первичной настройки.
- **Контекстный UI** — содержимое попапа зависит от активной вкладки (LinkedIn, rabota.by/hh.ru, Calendar, Meet, другая).

### LinkedIn — профиль и переписка

| Функция | Описание |
|---------|----------|
| **Виджет на странице профиля** | Блок с кнопками рядом с профилем (`linkedin.com/in/...`). |
| **Виджет в переписке** | Тот же блок в Messaging (`linkedin.com/messaging/...`); профиль определяется по открытому диалогу. |
| **Сохранение связи** | Поле ввода ссылки Huntflow + кнопка «Сохранить». Связь «профиль LinkedIn ↔ карточка Huntflow» сохраняется в HR Helper. |
| **Открыть в Huntflow** | Кнопка открывает карточку кандидата в Huntflow в новой вкладке. |
| **Копировать** | Копирование ссылки Huntflow в буфер обмена. |
| **Редактировать** | Показать поле ввода для смены/обновления ссылки Huntflow. |
| **Изменить статус** | Выбор статуса и причины отказа из Huntflow; обновление статуса кандидата. |

### Множественные вакансии (LinkedIn, rabota.by, hh.ru)

| Функция | Описание |
|---------|----------|
| **Несколько вакансий** | Если кандидат привязан к нескольким вакансиям, отображаются карточки по категориям: активные, отказы, нанятые. |
| **Карточки активных вакансий** | Название вакансии, дата (формат дд.мм.гггг), кнопки «Изменить статус», «В новый цикл» (при отказе). |
| **Карточки отказов** | Название вакансии, причина отказа, дата/время (дд.мм.гггг чч:мм), квадратная иконка перезапуска справа («В новый цикл»). |
| **Свёрнутые отказы** | При 2+ отказах — кнопка «▶ ❌ Отказы (N)» с переключением раскрытия/сворачивания. При одном отказе — обычное отображение. |
| **Бейдж «Сотрудник»** | При статусе «Нанят» — блокировка действий, отображение бейджа. |
| **В новый цикл** | Перевод rejected → new; работает с huntflow_url (rabota.by/hh.ru) и profileUrl (LinkedIn). |
| **+ Взять на другую вакансию** | Модальное окно выбора вакансии и добавление кандидата на неё. |
| **Скрытие блока статуса** | Блок «Изменение статуса / Причина» скрыт, если все вакансии — отказы (нет активных). |

### rabota.by и hh.ru — страницы резюме

| Функция | Описание |
|---------|----------|
| **Content script** | `content-resume.js` на страницах `/resume/...`. |
| **Привязка Huntflow** | Поле для ссылки Huntflow и сохранение связи «резюме ↔ Huntflow». |
| **Данные из Huntflow** | Подтягивание данных кандидата из Huntflow и отображение под заголовком резюме. |
| **Множественные вакансии** | Те же карточки и действия, что и для LinkedIn (status-multi по huntflow_url). |

### Google Calendar и Google Meet

| Функция | Описание |
|---------|----------|
| **Calendar** | Блок в попапе с кратким текстом о синхронизации; основная логика — в веб-приложении HR Helper. |
| **Meet** | Кнопки Scorecard, Контакт, Копировать текст уровня. Активны, когда в HR Helper есть данные по текущему звонку. |

### Технические детали (1.2.x)

- **Manifest 3**, service worker (`background.js`).
- **API-прокси** — все запросы к HR Helper идут через background; кэш 30 сек для `GET /status/` и `/status-multi/`.
- **Хранение** — `chrome.storage.sync` для baseUrl и apiToken.
- **Разрешения** — LinkedIn, Huntflow, HR Helper, rabota.by, hh.ru, Calendar, Meet, localhost.
